# Presentation Upgrade Agents — U-FORMA

## Что нашли на GitHub

- `hugohe3/ppt-master` — основной кандидат для editable PowerPoint: ~24.9K⭐, Python, делает реальные редактируемые PPTX, умеет работать с документами и шаблонами.
- `zarazhangrui/frontend-slides` — лучший дизайн-подход для сильных HTML/визуальных презентаций: ~20.6K⭐, zero-dependency HTML, анти-generic AI дизайн, умеет конвертацию/улучшение слайдов.
- `presenton/presenton` — self-hosted альтернатива Gamma/Beautiful.ai: ~8K⭐, Docker/API, editable PPTX export, но тяжелее для текущего VPS 1GB RAM.
- `icip-cas/PPTAgent` — исследовательский agentic framework: ~4.6K⭐, полезен как архитектурный референс, но менее практичен для быстрого продакшн-редактирования.

## Выбранный стек для нашей машины

1. **PPT Builder**: `python-pptx` + подход `ppt-master` — чтобы на выходе был редактируемый `.pptx`, а не картинка.
2. **Design Director**: правила `frontend-slides` — больше визуальной иерархии, меньше generic AI-slop, больше схем/цифр/карточек.
3. **QA Reviewer**: визуальный аудит через рендер/vision + проверка структуры PPTX.

## Главный агент

### Deck Orchestrator / Presentation CEO

**Задача:** управлять всей цепочкой улучшения презентации.

**Вход:** старый PDF/PPTX, цель, аудитория, стиль, правки владельца.

**Выход:** новая редактируемая презентация + PDF/preview + список правок.

**KPI:** презентация стала короче, сильнее продаёт, каждый слайд имеет один главный вывод, файл открываемый и редактируемый.

## Субагенты

### 1. Content Strategist

- Сокращает текст на 30–50%.
- Переводит документ из “технического описания” в коммерческое предложение.
- Усиливает оффер, выгоды, следующий шаг.

### 2. Design Director

- Задаёт стиль, палитру, типографику, визуальную систему.
- Следит, чтобы презентация не выглядела как шаблонный AI-документ.
- Делает разные типы слайдов: обложка, before/after, карточки, схема, таймлайн, тарифы, CTA.

### 3. Deck Builder

- Собирает `.pptx` как editable PowerPoint.
- Не вставляет слайды плоскими картинками.
- Сохраняет понятные текстовые блоки, фигуры, карточки, таблицы.

### 4. Commercial QA

- Проверяет, продаёт ли презентация.
- Находит слабые места: нет CTA, цены не выделены, слишком много текста, непонятен результат.

### 5. Technical QA

- Проверяет, что файл создан, открывается библиотекой, в нём нужное число слайдов, нет пустых слайдов.
- При наличии LibreOffice/рендера делает PDF/PNG-превью.

## Что сделали в v5

- Старый PDF: 6 страниц, много текста, одинаковая структура.
- Новый PPTX: 8 слайдов, редактируемый формат.
- Усилены: обложка, результат для клиента, агентная схема, операционный день, страница стоимости, CTA.
- Стиль: premium fashion operations — светлая база + deep ink + blush/sage/coral/gold accents.
