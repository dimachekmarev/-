# VK Publishing Contour (Semi-Auto)

## Цель
Стабильный ежедневный выпуск VK-контента с минимальным участием Дмитрия.

## Поток
1. RU Daily Pack -> черновик VK-поста
2. QA Agent -> проверка фактов/тона/CTA
3. Approval Gate -> подтверждение Дмитрия (только риск/цифры/оффер)
4. Ручная публикация в VK
5. Лог в KPI-щит

## Шаблон VK-поста
- Хук (1-2 строки)
- Боль SMB
- Практическое решение (3-5 пунктов)
- Ограничение/честный дисклеймер
- CTA в Telegram

## Approval required
- Есть конкретные цифры/обещания
- Упоминание клиентов/кейсов
- Новый оффер/прайс

## KPI по VK
- 1 пост/день
- ER по реакции/комментам
- Переходы в Telegram
- Входящие диалоги после поста
