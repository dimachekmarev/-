# SEO + скорость + доверие сайта Smart Agent AI

## Быстрый диагноз на 2026-06-08

Проверенная страница: https://smart-agent-ai.ru/articles/

Факты первичной проверки:
- HTML страницы: ~129 KB.
- Ресурсы на странице: 60 элементов.
- CSS: около 20 файлов.
- JS: около 18 файлов.
- Изображения: около 16.
- Заголовки отдают `Cache-Control: no-store, no-cache, must-revalidate` и `PHPSESSID` — это ломает нормальное кеширование публичной страницы.
- Есть mixed content (смешанный контент): одно изображение в HTML указано через `http://`, хотя сайт на `https://`.
- На странице много CSS/JS от WordPress, темы и Ibtana Visual Editor; часть стилей выглядит как редакторские/визуальные зависимости, которые могут грузиться на публичной странице лишними.
- Карточки статей используют 1024×1024 PNG — для списка статей это тяжело, нужно WebP/AVIF и правильные размеры.
- `robots.txt` указывает sitemap через `http://smart-agent-ai.ru/sitemap_index.xml`, нужно привести к `https://`.

## Цель

Сайт должен:
1. Открываться быстро на мобильном интернете.
2. Давать Google/Яндексу чистые технические сигналы.
3. Расти в доверии поиска: экспертность, структура, перелинковка, авторство, регулярность.
4. Не ломать публикацию статей и текущий WordPress-процесс.

## Приоритет 0 — измерить до исправлений

Ответственный: `worker-research`.

- [ ] Прогнать Lighthouse/PageSpeed для главной и `/articles/`.
- [ ] Зафиксировать Core Web Vitals (ключевые метрики скорости): LCP, CLS, INP/TBT, TTFB.
- [ ] Снять список самых тяжёлых ресурсов: CSS, JS, изображения, шрифты.
- [ ] Проверить мобильную версию: первый экран, меню, карточки статей.
- [ ] Проверить индексацию: robots, sitemap, canonical, schema.org, meta title/description.
- [ ] Сохранить отчёт в Obsidian рядом с этим чеклистом.

## Приоритет 1 — скорость загрузки

Ответственный: `worker-execution`.

- [ ] Включить кеширование публичных страниц WordPress: не отдавать `no-store` для обычных страниц.
- [ ] Разобраться, почему ставится `PHPSESSID` на публичной странице; убрать с анонимных посетителей, если не нужен.
- [ ] Включить сжатие Brotli/Gzip на HTML/CSS/JS/SVG.
- [ ] Включить браузерный кеш для картинок, CSS, JS, шрифтов.
- [ ] Убрать лишние CSS/JS, которые не нужны на `/articles/`.
- [ ] Проверить Ibtana Visual Editor: отключить публичные ассеты там, где блоки не используются.
- [ ] Отложить/деферить некритичный JS (defer/delay JS).
- [ ] Убрать `jquery-migrate`, если тема/плагины без него работают.
- [ ] Минифицировать и объединить CSS/JS без поломки верстки.
- [ ] Добавить critical CSS (критические стили первого экрана) или хотя бы убрать блокирующие стили.

## Приоритет 2 — изображения

Ответственный: `worker-execution`.

- [ ] Перевести обложки статей в WebP/AVIF.
- [ ] Для списка статей использовать не 1024×1024, а размер под карточку: примерно 320–600 px.
- [ ] Включить lazy loading (ленивая загрузка) для всех изображений ниже первого экрана.
- [ ] Первую важную картинку оставить с `fetchpriority="high"`, остальные — lazy.
- [ ] Исправить mixed content: все изображения и sitemap только через `https://`.
- [ ] Добавить понятные `alt` для ключевых изображений: не пустые и не спамные.

## Приоритет 3 — техническое SEO

Ответственный: `worker-content` + `worker-execution`.

- [ ] Исправить `robots.txt`: sitemap только `https://smart-agent-ai.ru/sitemap_index.xml`.
- [ ] Проверить sitemap: нет ли мусорных страниц, дублей, технических URL.
- [ ] Проверить canonical на главной, статьях и архивах.
- [ ] Проверить хлебные крошки (BreadcrumbList schema).
- [ ] Добавить Article schema для статей: author, datePublished, dateModified, image, headline.
- [ ] Добавить Organization schema: название, URL, логотип, контакты, социальные ссылки.
- [ ] Проверить Open Graph: title, description, image для шаринга.
- [ ] Проверить 404, редиректы, www/non-www, http→https.

## Приоритет 4 — доверие поиска / E-E-A-T

E-E-A-T = опыт, экспертность, авторитетность, доверие.

Ответственный: `worker-content`.

- [ ] Добавить страницу «О компании» с конкретикой: кто мы, что внедряем, для кого, как работаем.
- [ ] Добавить страницу автора/эксперта: Дмитрий Чекмарев, опыт, проекты, контакты.
- [ ] В каждой статье показывать автора, дату публикации и дату обновления.
- [ ] Добавить блок «Проверено/обновлено» для важных материалов.
- [ ] Добавить реальные кейсы/разборы: задача → решение → результат → ограничения.
- [ ] Добавить контакты и понятный путь заявки: Telegram/форма/почта.
- [ ] Добавить политику конфиденциальности, условия, реквизиты/контактные данные.
- [ ] Перелинковывать статьи между собой: 3–5 внутренних ссылок в каждой статье.
- [ ] Сделать тематические кластеры: AI-продажи, AI-поддержка, CRM, контент, документы.

- [ ] Для каждой группы сделать landing/article map (какая страница отвечает за какой запрос).

- [ ] Добавить FAQ-блоки с микроразметкой там, где уместно.
- [ ] Настроить еженедельный SEO-аудит: новые статьи, позиции, индексация, ошибки.

## Контроль качества

Ответственный: `controller-quality`.

Перед закрытием задачи проверить:
- [ ] Страница `/articles/` стала кешируемой.
- [ ] Нет `http://` ресурсов внутри `https://` страницы.
- [ ] HTML стал легче или хотя бы не вырос.
- [ ] CSS/JS стало меньше или они отложены.
- [ ] Изображения оптимизированы.
- [ ] Sitemap и robots отдают правильные `https://` ссылки.
- [ ] Lighthouse/PageSpeed улучшился относительно базового замера.
- [ ] Ничего не сломалось визуально на мобильном.
