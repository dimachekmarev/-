# Личный бренд: автономная мультиканальная система

## Цель
Рост личного бренда Дмитрия как founder-практика AI-автоматизации: органический охват -> доверие -> входящие диалоги -> клиенты.

## Каналы (приоритет)
1. Instagram (основной рост/охват)
2. Telegram (доверие/прогрев)
3. VK + другие площадки (репликация лучших форматов)
4. GitHub (доказательство экспертности и кейсы)

## Принцип управления
- Агентная система делает 90% рутины.
- Дмитрий подтверждает только критичные шаги: публикации, офферы, спорные тезисы.

## Архитектура агентов
### L0 — Контроль
- Brand Orchestrator: план недели, приоритеты, распределение задач.
- QA Supervisor: проверка стиля, фактов, анти-клише, риск-контроль.

### L1 — Производство
- Idea Scout: собирает идеи из GitHub/проектов/голосовых заметок.
- Script Writer IG: Reels/карусели/сторис сценарии.
- Telegram Writer: посты, серии, закрепы, CTA.
- Repurposer: один инсайт -> 4-6 форматов под разные платформы.

### L2 — Дистрибуция
- Publisher Queue: календарь публикаций, очередь, статусы.
- Community Drafts: черновики ответов на комментарии/личку.
- Performance Analyst: ежедневный отчёт + weekly рекомендации.

## Approval gates (что подтверждает Дмитрий)
1. Недельная тема и оффер
2. Публикации с обещаниями/цифрами/кейсами клиентов
3. Изменение позиционирования
4. Платная воронка и офферы

## Контент-ядро (4 столпа)
1. Build in public (что строим и почему)
2. Кейсы автоматизации SMB (проблема -> решение -> результат)
3. Системные разборы (архитектура, ошибки, ограничения)
4. Founder log (личный опыт, решения, антихайп)

## KPI
### 30 дней
- Единый контент-ритм: 5-7 выходов/нед
- Стабильный weekly recap
- Рост входящих диалогов в Telegram

### 90 дней
- Повторяемый контент-конвейер
- Рост органического охвата на ключевых платформах
- Регулярные целевые запросы от бизнеса

## Операционный цикл
- Пн: стратегия недели + контент-пакет
- Вт-Чт: публикации + репурпозинг
- Пт: кейс/результат + CTA
- Вс: аналитика + корректировка

## Технический план внедрения

### Этап 2 (сейчас)

## Риски и защита

- Без «магических обещаний»
- Факт-чекинг перед публикацией
- Журнал решений в Obsidian
