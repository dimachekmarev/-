# Phase 1 Money Room — outreach scripts

Offer: **AI-бот для приёма заявок за 48 часов**.

Status: **draft only**. Ничего не отправлять клиентам до approval от Дмитрия / revenue-controller и QA go/no-go.

## 0. Правила перед любым касанием

1. Открыть публичный источник лида и вручную подтвердить, что бизнес живой и канал обращения актуален.
2. Не использовать приватные телефоны/email/имена, если они не были явно approved для outreach.
3. Не обещать продажи, мед/фин консультации, сложные CRM-интеграции или «замену отдела продаж».
4. CTA всегда короткий: **«показать пример сценария / 7 минут разбор заявки»**, не длинный созвон.
5. Если собеседник просит подробности — переводить в demo/proposal lane, а не слать простыню.

## 1. Переменные для персонализации

Использовать 1–2 переменные, не больше:

- `{{company}}` — название компании из tracker.
- `{{niche}}` — ниша: стоматология, клиника, косметология, автоуслуги, ремонт, онлайн-школа.
- `{{pain}}` — короткая гипотеза боли из lead list.
- `{{scenario}}` — пример сценария: «услуга → 3–5 вопросов → удобное время → заявка администратору».
- `{{source_hint}}` — что увидели на публичной странице: онлайн-запись, WhatsApp/Telegram, форма заявки, VK.

Не писать: «я изучил ваш бизнес глубоко», если проверка была только по публичной странице.

## 2. Первое касание #1 — мягкое

Цель: безопасно открыть диалог, без давления и без ощущения массовой рассылки.

```text
Здравствуйте. Увидел у {{company}} {{source_hint}} — похоже, часть заявок приходит через сообщения/форму.

Я делаю короткого AI-бота для приёма заявок: он отвечает быстро, задаёт 3–5 вопросов, собирает контакт и передаёт администратору уже понятную заявку.

Для {{niche}} можно показать простой сценарий: {{scenario}}.
Показать пример в 2–3 сообщениях?
```

Короткая версия для Telegram/VK:

```text
Здравствуйте. У {{company}} вижу заявки через {{source_hint}}. Делаю AI-бота, который за 48 часов можно поставить на приём заявок: 3–5 вопросов, контакт, передача администратору. Показать короткий пример под {{niche}}?
```

Когда использовать:
- первый контакт с score 4–5;
- медицинские/клинические ниши, где нужен аккуратный тон;
- когда нет явной боли, только публичный канал заявки.

## 3. Первое касание #2 — прямое

Цель: быстро проверить интерес у владельца/администратора, когда боль очевидна.

```text
Здравствуйте. Быстрый вопрос по заявкам {{company}}.

Если клиент пишет вечером или задаёт типовые вопросы до записи, часть обращений может остывать до ответа администратора.

Я запускаю за 48 часов AI-бота, который:
— отвечает сразу;
— уточняет услугу/время/контакт;
— передаёт заявку вам в Telegram/таблицу/CRM-ready формате.

Без замены администратора и без сложной интеграции.
Показать демо-сценарий на 7 минут?
```

Короткая версия:

```text
Здравствуйте. Помогаю не терять заявки, когда клиент пишет не в момент ответа администратора. За 48 часов ставлю AI-бота: быстрый ответ, 3–5 вопросов, контакт, заявка менеджеру. Показать демо-сценарий на 7 минут?
```

Когда использовать:
- автоуслуги, ремонт, окна/потолки;
- лиды с WhatsApp/Telegram/онлайн-формой;
- после ручной проверки, что у бизнеса есть поток входящих.

## 4. Первое касание #3 — экспертное

Цель: выглядеть не как «продажа бота», а как быстрый аудит узкого участка воронки.

```text
Здравствуйте. Смотрю на заявки в {{niche}}: обычно теряются не все лиды, а конкретный участок — первый ответ и сбор нормального брифа.

Для {{company}} я бы проверил такой сценарий:
1) клиент выбирает услугу;
2) бот задаёт 3–7 вопросов без консультаций и диагнозов;
3) администратор получает уже структурированную заявку;
4) дальше человек спокойно доводит до записи/расчёта.

Такой MVP можно собрать за 48 часов, если есть тексты услуг и ответственный менеджер.
Хотите, покажу, как это выглядело бы на вашем примере?
```

Короткая версия:

```text
Здравствуйте. В {{niche}} часто проседает не реклама, а первый ответ: клиент написал, но до нормального брифа не дошёл. Я могу показать мини-сценарий AI-бота для {{company}}: услуга → 3–7 вопросов → заявка администратору. Показать пример?
```

Когда использовать:
- score 5;
- клиники, стоматологии, онлайн-школы;
- когда нужно подчеркнуть безопасность: бот не консультирует, а только собирает заявку.

## 5. 48h follow-up sequence

### H0 — первое касание

Отправить один из трёх approved скриптов. В CRM поставить:

- `status = contacted`
- `last_touch_date = today`
- `next_action_due = +3h` для score 5, `+24h` для score 4
- `notes = script_soft/direct/expert + personalization token`

### H3–H4 — быстрый follow-up для score 5 / горячих ниш

```text
Уточню коротко: я не предлагаю «заменить администратора».
Идея проще — не дать заявке пропасть до ответа человека: бот собирает контакт, услугу и удобное время.

Могу показать мини-демо на вашем сценарии без подготовки с вашей стороны. Актуально?
```

Если ниша медицина/стоматология:

```text
Важное уточнение: бот не даёт медицинских консультаций и не ставит диагнозы. Он только собирает заявку и передаёт администратору: услуга, удобное время, контакт, комментарий.
Показать безопасный пример такого сценария?
```

### H24 — follow-up без давления

```text
Здравствуйте. Вернусь один раз по теме заявки.

Если сейчас не до внедрений, можно начать с малого: я покажу 1 сценарий приёма заявки для {{niche}} — что спрашивать, куда передавать, где не обещать лишнего.

Если после примера будет видно, что пользы нет — спокойно закрываем вопрос.
Показать?
```

### H36 — follow-up с конкретным next step

```text
Есть быстрый формат: 7 минут разбора.
Я беру одну типовую заявку {{niche}} и показываю путь:
клиент → 3–5 вопросов → контакт → заявка администратору.

Если подходит, дальше можно запустить MVP за 48 часов после материалов. Если нет — без обязательств.
Когда удобнее показать: сегодня или завтра?
```

### H48 — закрывающий follow-up / exit

```text
Похоже, сейчас не приоритет — окей.

Оставлю коротко: если заявки из {{source_hint}} иногда теряются или приходят вне рабочего времени, можно быстро поставить MVP-бота на сбор заявок без сложной интеграции.

Если станет актуально — напишите «бот», пришлю пример сценария под {{niche}}.
```

CRM после H48 без ответа:

- `status = nurture`
- `next_action_due = +14d`
- `reply_summary = no_reply_after_48h_followup`

## 7. Objection handling

### «Дорого»

```text
Понимаю. Тогда не предлагаю сразу внедрение.

Можно начать с мини-аудита за 4 900 ₽: покажу, где в текущем пути заявки есть потери, и дам схему бота на 1 сценарий. Если потом решите запускать — зачтём аудит в стоимость.

Так безопаснее, чем сразу брать большой пакет.
```

Если нельзя предлагать аудит без approval цены:

```text
Понимаю. Тогда сначала покажу бесплатный короткий пример сценария. Если увидите пользу — обсудим минимальный Start без лишних функций.
```

### «У нас уже есть администратор»

```text
Это как раз нормально. Бот не заменяет администратора.

Он снимает с него первый однотипный шаг: принять сообщение, уточнить услугу/время/контакт и передать готовую заявку. Дальше отвечает человек.

Так администратор меньше вытягивает данные вручную и быстрее берёт тёплую заявку.
```

### «У нас уже есть CRM / форма / виджет»

```text
Отлично, значит процесс уже есть.

Я не предлагаю ломать CRM. MVP можно поставить перед CRM или рядом: собрать нормальную заявку и передать её в удобном виде. Интеграции трогаем только после проверки, что сценарий реально даёт пользу.
```

### «Бот будет раздражать клиентов»

```text
Да, если сделать длинный и навязчивый сценарий.

Поэтому MVP короткий: 3–5 вопросов, возможность перейти к человеку, без давления и без «умных» ответов там, где нужен администратор.

Главная задача — быстрее принять заявку, а не вести бесконечный чат.
```

### «Нам не нужен AI»

```text
Понимаю. Я бы не продавал это как AI ради AI.

Суть не в технологии, а в результате: клиент получил быстрый первый ответ, оставил данные, администратор получил нормальную заявку. Если это уже работает без потерь — внедрение не нужно.

Могу показать сценарий, чтобы вы сами оценили, есть ли смысл.
```

## 8. Нишевые углы персонализации

### Стоматологии

- Боль: дорогой лид, запись на консультацию/имплантацию/гигиену, вечерние сообщения.
- Важно: не обещать диагностику или медицинские советы.
- Сценарий: услуга → жалоба/цель визита → удобное время → контакт → администратор.

Фраза:

```text
Для стоматологии бот не консультирует по лечению — только помогает быстро собрать заявку на запись и передать её администратору.
```

### Частные клиники

- Боль: много типовых вопросов до записи, чувствительность к медобещаниям.
- Важно: «без медконсультаций».
- Сценарий: направление/услуга → город/филиал → удобное время → контакт.

### Косметология / салоны / барбершопы

- Боль: запись в директ/мессенджер, администратор занят, клиент выбирает быстро.
- Сценарий: услуга → мастер/время → контакт → запись/уточнение.

### Автоуслуги

- Боль: расчёт стоимости, фото/описание проблемы, срочность.
- Сценарий: услуга → марка/модель → проблема/фото → удобное время → заявка мастеру.

### Ремонт / окна / потолки

- Боль: заявки на расчёт/замер, нужно быстро собрать параметры.
- Сценарий: объект → площадь/адрес район → что нужно → удобное время → заявка замерщику.

### Онлайн-школы

- Боль: лиды с рекламы/пробного урока, много повторяющихся вопросов.
- Сценарий: курс → уровень/цель → удобное время → контакт → менеджер/пробный урок.

## 10. Рекомендуемый старт после approval

1. Не отправлять всем 120 сразу.
2. Первая волна: 20 лидов score 5 из Segment A.
3. Расклад текстов:
   - 8 мягких — медицина/стоматология;
   - 8 прямых — авто/ремонт/салоны;
   - 4 экспертных — score 5 с явным каналом заявки.
4. После 20 касаний ждать первые ответы и не масштабировать, если текст даёт <2 ответов.
5. После 30 касаний при <3 ответах — менять первый абзац и CTA, не добивать тем же текстом.
