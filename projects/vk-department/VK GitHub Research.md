# VK GitHub Research

Цель: взять самое полезное из GitHub-экосистемы для VK, но не тащить лишний код на VPS 1GB RAM.

## Отобранные репозитории

### 1. python273/vk_api
- GitHub: https://github.com/python273/vk_api
- Язык: Python
- Примерная роль: классический Python-wrapper для VK API.
- Полезно для: загрузки фото, wall.post, messages, bots longpoll.
- Решение: держать как основной кандидат, если текущий raw API-скрипт станет большим.

### 2. vkbottle/vkbottle
- GitHub: https://github.com/vkbottle/vkbottle
- Язык: Python
- Примерная роль: асинхронный фреймворк для VK-ботов.
- Полезно для: отдельного обработчика входящих сообщений и воронок.
- Решение: использовать позже, когда будем делать постоянного VK-бота/оператора.

### 3. negezor/vk-io
- GitHub: https://github.com/negezor/vk-io
- Язык: TypeScript / Node.js
- Примерная роль: современный VK SDK для Node.
- Полезно для: будущей панели управления, если фронт/бэкенд будет на Node.
- Решение: сейчас не ставить, чтобы не раздувать сервер.

### 4. prostomarkeloff/vkwave
- GitHub: https://github.com/prostomarkeloff/vkwave
- Язык: Python
- Примерная роль: асинхронная работа с VK API.
- Полезно для: высоконагруженных сообщений/ботов.
- Решение: резервный вариант для масштабирования.

### 5. VKCOM/vk-api-schema
- GitHub: https://github.com/VKCOM/vk-api-schema
- Роль: официальная схема VK API.
- Полезно для: проверки методов, полей и типов.
- Решение: использовать как справочник.

### 6. Bizordec/vk-to-tgm
- GitHub: https://github.com/Bizordec/vk-to-tgm
- Роль: мост VK → Telegram.
- Полезно для: идей по пересылке постов/стены в Telegram.
- Решение: изучить позже для моста лидов и уведомлений.

## Архитектурное решение на сейчас
Не клонировать и не ставить всё подряд. Для текущей задачи лучше:
- lightweight (лёгкий) raw VK API через один Python-скрипт;
- собственный state.json против повторов;
- LLM генерирует текст, скрипт публикует и возвращает ссылку;
- позже добавить vkbottle для постоянной обработки входящих сообщений.

## Почему так
На VPS 1GB RAM нельзя плодить тяжёлые сервисы. Нужен устойчивый минимум: постинг, картинки, ссылки, логи, без лишних демонов.
