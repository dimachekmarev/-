# Анализ: агентная система производства и публикации видео

Дата анализа: 2026-06-13
Референс: [Claude + Ютуб = $30,000/Месяц](https://youtu.be/1oAzhMskoG0) — Артем Слабун

---

## 1. Стержневая модель референс-видео

### Ниша
- «Первобытные люди» / история / как жили раньше
- Годится любая «объяснительная» ниша: еда, средневековье, будущее, космос, фантастика

### Инструменты
| Инструмент | Роль |
|---|---|
| **Claude Code (desktop)** | мозг: пишет сценарий, рулит генерацией, скачивает файлы |
| **Higgsfield MCP** | навык внутри Claude: генерирует картинки и анимацию |
| **vidIQ (расширение)** | вытягивает текст/сценарий конкурентов с тайм-кодами |
| **NextLev (расширение)** | экспорт сценария файлом (если vidIQ глючит) |
| **Любой видеоредактор** | сборка картинок по тайм-кодам + наложение озвучки |

### Рабочий процесс (полный цикл одного видео)
1. **Сбор сценариев конкурентов** → vidIQ/NextLev, копировать с тайм-кодами
2. **Загрузка в Claude** + мастер-промт («проанализируй, напиши новый сценарий, убери ИИ-шность, сделай живым»)
3. **Claude выдаёт сценарий** с покадровыми тайм-кодами + описанием визуала к каждому кадру
4. **Ручная вычитка** в переводе на русский → правки человечности → обратно на английский (для озвучки)
5. **Генерация картинок** через Higgsfield MCP: Claude сам входит в аккаунт, сам генерирует 100+ кадров
6. **Скачивание** картинок с именами по тайм-коду (000.jpg, 003.jpg, 006.jpg …)
7. **Анимация выборочных кадров** (каждый 5-й или первые 30 сек) — чтобы избежать флага «шаблонный контент»
8. **Монтаж** в редакторе: дорожка озвучки + картинки подгоняются по имени/секундам
9. **Публикация** (в ролике не показана, только implied)

### Ключевая фишка
Вся цепочка управляется **одним диалогом** в Claude Code. Один промт = сценарий + генерация всех кадров + скачивание.

### Скорость
- Заявлено: < 20 минут на видео (без учёта рендера и вычитки)
- Реально для новичка: 40–90 минут с учётом правок

### Монетизация
- YouTube AdSense: канал из 7 видео → ~$30k/мес (по vidIQ, совпадает с реальной аналитикой автора)
- RPM в таких нишах (первобытные, история) выше среднего
- Множество «свежих» каналов в этой нише также делают деньги — не исключение

### Ограничения/нюансы, озвученные автором
- Бесплатный Claude имеет сильное ограничение по запросам → нужна подписка ($20/мес)
- Higgsfield MCP надо настраивать один раз через терминал (3 команды)
- Каждый сценарий обязательно вычитывать человеком — чистый ИИ-текст «невозможно слушать»
- Без анимации YouTube может отобрать монетизацию за «шаблонный контент»

## 2. «Сделать так же» для Дмитрия (адаптация, не копирование)

### Что берём
- **Конвейерную архитектуру**: анализ доноров → сценарий → визуал → озвучка → монтаж → публикация
- **Паттерн «один диалог управляет всем»** — Claude/Hermes как оркестратор
- **Сбор сценариев конкурентов с тайм-кодами** как топливо для генерации
- **Промт-шаблоны** для сценариев и генерации визуала (адаптировать под ниши Дмитрия)
- **Человеческую редактуру** как обязательный гейт качества
- **Выборочную анимацию** для прохождения политик YouTube

### Что НЕ копируем слепо
- Нишу «первобытные» — у Дмитрия свои ниши из Version 1: психология, отношения, деньги, привычки
- Higgsfield как единственный генератор — нужен свой стек (см. Блокеры)
- Англоязычную озвучку — у Дмитрия русскоязычный контент (пока)
- Ставку только на YouTube AdSense — у Дмитрия цель: TG-воронка + услуги + партнёрки

### Ключевое отличие Дмитрия от референса
Артем Слабун делает **educational/исторический** контент с простыми картинками. Дмитрий делает **экспертный/личный** контент (психология, деньги, привычки), где голос и лицо (или аватар) важнее простых картинок.

## 3. Команда агентов (расширенная архитектура)

### Агент 1: Research/Donor Agent
**Вход:** ниша, ключевые слова, список конкурентов
**Выход:** карточки доноров по формату реестра + дайджест трендов на день
**Артефакт:** `donors/donor-YYYY-MM-DD.json` + `trends/daily-trends.md`
**KPI:** не менее 5 новых доноров/неделя; тренд-дайджест каждое утро
**Гейт приёмки:** у каждого донора заполнены: ссылка, ниша, формат, что берём, риск

### Агент 2: Niche/Channel Strategist
**Вход:** данные доноров, метрики канала, KPI доски
**Выход:** выбор 2–3 фокус-ниш на неделю + решение о форматах (shorts/long/mixed)
**Артефакт:** `strategy/weekly-niche-focus.md`
**KPI:** CTR > 4% по выбранным нишам; рост retention неделя к неделе
**Гейт приёмки:** ниши не пересекаются, под каждую есть 5+ доноров

### Агент 3: Script Agent
**Вход:** тема от Research Agent + сценарии доноров (с тайм-кодами) + Tone-of-Voice Дмитрия
**Выход:** сценарий с покадровыми тайм-кодами + описаниями визуала + пометкой «ручная вычитка»
**Артефакт:** `scripts/script-YYYY-MM-DD-topic.md`
**KPI:** выход ≥ 2 сценариев/день; ≤ 30% правок человеком после первой итерации
**Гейт приёмки:** сценарий прочитан человеком вслух, неестественные фразы исправлены, хронометраж ±20% цели

### Агент 4: Visual/Storyboard Agent
**Вход:** сценарий с описаниями визуала + референсы стиля
**Выход:** сгенерированные кадры (имена по тайм-кодам) + 15–20% анимированных
**Артефакт:** `assets/video-YYYY-MM-DD/frames/` + `assets/video-YYYY-MM-DD/animations/`
**KPI:** покрытие ≥ 90% тайм-кодов кадрами; стиль выдержан в одном ключе
**Гейт приёмки:** визуал соответствует тексту диктора в каждом кадре, стиль един

### Агент 5: Voiceover/Audio Agent
**Вход:** финальный сценарий (после ручной вычитки) + голосовой профиль
**Выход:** аудио-дорожка (wav/mp3) + субтитры (.srt)
**Артефакт:** `assets/video-YYYY-MM-DD/voiceover.mp3` + `subtitles.srt`
**KPI:** озвучка готова за ≤ 15 минут; ≤ 2 ошибок произношения на минуту
**Гейт приёмки:** естественность голоса, паузы в правильных местах, синхронизация субтитров

### Агент 6: Video Generation/Editor Agent
**Вход:** кадры + анимации + озвучка + субтитры
**Выход:** смонтированное видео (MP4) с субтитрами и фоновой музыкой (опционально)
**Артефакт:** `output/video-YYYY-MM-DD-final.mp4`
**KPI:** рендер ≤ 2× хронометража видео; сборка по скрипту без ручных правок монтажа
**Гейт приёмки:** видео проигрывается без ошибок, тайминг кадров совпадает с озвучкой, субтитры синхронизированы

### Агент 7: Packaging/SEO Agent
**Вход:** готовое видео + метаданные (ниша, ключевые слова, CTA)
**Выход:** 3–5 вариантов заголовков + превью (thumbnail) + описание + теги + хештеги
**Артефакт:** `packaging/video-YYYY-MM-DD-seo.md` + `thumbnails/thumb-*.png`
**KPI:** CTR опубликованного > 4% через 48 часов
**Гейт приёмки:** заголовки A/B протестированы (если возможно), превью привлекает внимание без кликбейта

### Агент 8: Publisher Agent
**Вход:** видео + SEO-пакет + расписание
**Выход:** опубликованное видео на YouTube + лог публикации
**Артефакт:** запись в `publish-log.md` с URL, временем, статусом
**KPI:** публикация в заданное время ± 15 минут; 0 ошибок загрузки
**Гейт приёмки:** видео доступно по ссылке, все метаданные на месте, превью отображается

### Агент 9: Analytics Feedback Agent
**Вход:** YouTube Analytics API / vidIQ данные
**Выход:** ежедневный отчёт + рекомендации на завтра (какие темы/форматы усилить/убрать)
**Артефакт:** `analytics/daily-report-YYYY-MM-DD.md`
**KPI:** отчёт ≤ 09:00 каждый день; рекомендации подтверждены метриками
**Гейт приёмки:** отчёт содержит CTR, retention, просмотры, лайки, подписки, лиды и конкретные действия

### Агент 10: Controller / Quality Gate
**Вход:** выходы всех агентов
**Выход:** решение GO/NO-GO по каждому этапу + сводный вечерний отчёт
**Артефакт:** `control/daily-gate-log.md`
**KPI:** 100% видео проходят через гейт; ≤ 5% брака после публикации
**Гейт приёмки:** видео соответствует политике YouTube, нет шаблонного контента, метаданные корректны

## 6. Блокеры и что нужно для публикации

### 1. YouTube OAuth / доступ к каналу
**Блокер:** Публикация видео через API требует OAuth 2.0 с доступами `youtube.upload`, `youtube.force-ssl`.
**Нужно:**
- Google Cloud Project с включённым YouTube Data API v3
- OAuth 2.0 Client ID (Desktop app или Web)
- Токен доступа (access + refresh) с областями:
  - `https://www.googleapis.com/auth/youtube.upload`
  - `https://www.googleapis.com/auth/youtube`
  - `https://www.googleapis.com/auth/yt-analytics.readonly`
**Статус в проекте:** [ ] Не подключено (судя по 90-day board)

### 2. Стек генерации видео
**Блокер:** Артем использует Higgsfield + Claude Code. У Дмитрия:
- Если Hermes Agent: нужен свой MCP-навык для генерации/анимации или интеграция с внешним API
- Варианты:
  - Higgfield API напрямую (нужен API ключ, регистрация)
  - RunwayML / Kaiber / другие video generation API
  - Midjourney (через бота/API) для статики + Runway для анимации
  - ComfyUI (self-hosted) для полного контроля
- **Рекомендация:** Начать с Midjourney (картинки) + RunwayML (анимация) как наиболее зрелые

### 3. Голос / озвучка
**Блокер:** Озвучка для русского языка. Референс использует ElevenLabs (подтверждено из контекста ниши).
**Нужно:**
- ElevenLabs API ключ (есть русские голоса) — платный, ~$5–$22/мес
- Альтернативы: SaluteSpeech (Сбер), Yandex SpeechKit, Silero TTS (локально)
- Для англоязычных роликов: ElevenLabs или Play.ht

### 4. Клипы / изображения / анимация
**Блокер:** Higgsfield имеет ограничения бесплатного тарифа. Нужно понять стоимость генерации 100+ кадров на видео.
- Higgsfield pricing: нужен анализ (кредиты/подписка)
- Альтернативы: DALL-E 3, Leonardo.ai, Playground AI

### 5. Политика одобрения (Approval Policy)
**Блокер:** Кто и когда принимает решение «публиковать»?
- Автоматический пайплайн без человека = риск публикации брака
- Нужен явный approval gate перед Publisher Agent:
  - Script → человек читает
  - Voiceover → человек слушает
  - Video → человек смотрит (можно не полностью)
  - Packaging → человек проверяет заголовок/превью
  - Публикация → человек нажимает «ОК» (или автоматически по расписанию после всех гейтов)

### 6. YouTube Policy Compliance
- Без анимации хотя бы 20% кадров → риск demonetization за «шаблонный контент»
- Заимствованный контент → risk of copyright strikes
- Нужен watermark / уникальный стиль для каждого канала

## 10. Резюме: отличие проекта Дмитрия от референса

| Аспект | Референс (Артем Слабун) | Дмитрий |
|---|---|---|
| Ниша | Первобытные/история (educational) | Психология/отношения/деньги (экспертный) |
| Контент | Простые картинки + озвучка | Возможно: лицо/аватар + графика + кейсы |
| Язык | EN (английский) | RU + возможно EN |
| Монетизация | Только YouTube AdSense | AdSense + TG-воронка + услуги + партнёрки |
| Оркестратор | Claude Code | Hermes Agent |
| Стек визуала | Higgsfield (один инструмент) | Midjourney + RunwayML / ComfyUI |
| Скорость | < 20 мин/видео | Цель: < 90 мин/видео (реалистично) |
| Команда | 1 человек + Claude | 1 человек + 10 агентов |

Главный вывод: **архитектура рабочая и подходит**. Принципиальных различий нет — адаптация состоит в замене ниш, языка и стека визуала. Самое узкое место сейчас — отсутствие YouTube OAuth доступа. Без него Publisher Agent не сможет публиковать, и вся система останется «на бумаге».
