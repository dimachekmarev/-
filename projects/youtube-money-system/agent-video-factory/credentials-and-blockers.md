# Доступы, инструменты и блокеры

## Что нужно Дмитрию предоставить/настроить

### 1. YouTube OAuth / доступ к каналу
- **Что:** Google Cloud Project с YouTube Data API v3, OAuth 2.0 Client ID (Desktop/Web), токен доступа.
- **Области:** `youtube.upload`, `youtube`, `yt-analytics.readonly`.
- **Статус:** [ ] НЕ ПОДКЛЮЧЕНО — блокер #1.
- **Действие Дмитрия:** создать проект в Google Cloud Console, включить API, получить Client ID и секрет, пройти OAuth flow, передать токен.

### 2. Сервис озвучки (TTS)
- **Что:** ElevenLabs API ключ (русские голоса) или аналог (SaluteSpeech, Yandex SpeechKit, Silero TTS).
- **Стоимость:** ElevenLabs ~$5–$22/мес.
- **Статус:** [ ] НЕ НАСТРОЕНО.
- **Действие Дмитрия:** зарегистрироваться в ElevenLabs, создать API ключ, выбрать голос.

### 3. Генератор картинок
- **Варианты:** Midjourney (через Discord/API), DALL-E 3 (OpenAI), Higgsfield, Leonardo.ai.
- **Рекомендация:** начать с Midjourney — лучший контроль стиля.
- **Статус:** [ ] НЕ ВЫБРАН.
- **Действие Дмитрия:** выбрать сервис, зарегистрироваться, получить API доступ или настроить бота.

### 4. Анимация кадров
- **Варианты:** RunwayML, Kaiber, ComfyUI (self-hosted).
- **Требование:** ≥ 15% кадров должны быть анимированы (политика YouTube).
- **Статус:** [ ] НЕ ВЫБРАН.
- **Действие Дмитрия:** выбрать сервис, получить API ключ.

### 5. Инструмент сбора сценариев конкурентов
- **Что:** vidIQ (расширение Chrome) или NextLev.
- **Статус:** [ ] НЕ УСТАНОВЛЕНО.
- **Действие Дмитрия:** установить расширение в браузер.

### 6. Оркестратор (мозг системы)
- **Что:** Claude Code / Hermes Agent с MCP-навыками.
- **Стоимость:** Claude Pro $20/мес (если Claude Code).
- **Статус:** [ ] НЕ НАСТРОЕНО для видео-конвейера.
- **Действие Дмитрия:** установить Claude Code desktop или настроить Hermes Agent с видео-MCP.

### 7. Telegram Bot (для CTA и лидов)
- **Что:** Bot Token от @BotFather.
- **Статус:** [ ] ВЕРОЯТНО ЕСТЬ, но не интегрирован в конвейер.
- **Действие Дмитрия:** подтвердить/создать бота, передать токен для интеграции в CTA.

### 8. Политика одобрения (Approval Policy)
- **Что:** явные правила: кто и когда нажимает «публиковать».
- **Предложение:**
  - Script → человек читает → OK
  - Voiceover → человек слушает → OK
  - Video → человек смотрит (можно не полностью) → OK
  - Packaging → человек проверяет заголовок/превью → OK
  - Публикация → Controller ставит GO (человек может отменить)
- **Статус:** [ ] НЕ УТВЕРЖДЕНО.
- **Действие Дмитрия:** утвердить или скорректировать цепочку одобрения.
