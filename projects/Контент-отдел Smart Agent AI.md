# Контент-отдел Smart Agent AI + Личный бренд

## 1. Short Verdict

Автоматизировать в первую очередь: **контент-конвейер из сырья в посты**. Голосовые заметки и результаты проектов → структурированный контент → адаптация под площадки → очередь на публикацию. Human-in-the-loop на этапе финального утверждения.

Не автоматизировать первым: прямую публикацию без approval, генерацию фактов без источника, обещания от лица компании.

## 3. Process Lanes

### Lane 1: Генерация идей / сбор сырья
- **Owner:** Дмитрий
- **Trigger:** появилась мысль / завершён проект / интересная новость
- **Input:** голосовое сообщение, заметка, лог проекта
- **Steps:** наговорить → расшифровать → структурировать → сохранить в базу идей
- **Tools:** Telegram (голосовые), Whisper (транскрибация), Obsidian (хранение)
- **Output:** структурированная заметка в базе идей
- **Frequency:** ежедневно
- **Pain:** идеи теряются, если не записать сразу

### Lane 2: Производство контента
- **Owner:** Content Ops Agent
- **Trigger:** новая заметка в базе идей
- **Input:** сырая заметка, контекст проекта
- **Steps:** взять идею → определить формат → написать черновик → адаптировать под площадки
- **Tools:** LLM (DeepSeek), Obsidian, Canva API (изображения)
- **Output:** черновики постов для каждой площадки
- **Frequency:** ежедневно
- **Pain:** ручное написание текстов, адаптация под форматы

### Lane 3: Визуальный контент
- **Owner:** Visual Agent
- **Trigger:** готовый текст поста
- **Input:** текст, тема, бренд-гайд
- **Steps:** сгенерировать изображение → проверить соответствие бренду → прикрепить к посту
- **Tools:** NanoBanana / ComfyUI / Stable Diffusion (генерация), Canva (шаблоны)
- **Output:** изображение/карусель/обложка
- **Frequency:** с каждым постом
- **Pain:** отдельный процесс, требует дизайна

### Lane 4: Публикация и планирование
- **Owner:** Publisher Agent
- **Trigger:** утверждённый пост
- **Input:** готовый текст + изображение + площадка
- **Steps:** поставить в очередь → опубликовать в указанное время → залогировать
- **Tools:** Telegram API, X API, VK API, LinkedIn API, Buffer/Notion (контент-план)
- **Output:** опубликованный пост, запись в контент-плане
- **Frequency:** по расписанию (1-3 раза в день)
- **Pain:** рутина публикации, тайминг по разным часовым поясам

### Lane 5: Аналитика и обратная связь
- **Owner:** Analytics Agent
- **Trigger:** ежедневно/еженедельно
- **Input:** метрики всех площадок
- **Steps:** собрать охваты/реакции/переходы → сравнить с прошлым периодом → дать рекомендации
- **Tools:** API платформ, Google Analytics
- **Output:** еженедельный отчёт
- **Frequency:** еженедельно
- **Pain:** разрозненные данные, нет единой картины

### Lane 6: Вовлечение и комьюнити
- **Owner:** Community Agent
- **Trigger:** новый комментарий / сообщение
- **Input:** комментарий, вопрос, упоминание
- **Steps:** классифицировать → подготовить драфт ответа → эскалировать сложное человеку
- **Tools:** API платформ
- **Output:** драфт ответа или эскалация
- **Frequency:** по мере поступления
- **Pain:** ручной мониторинг всех площадок

### Lane 7: База знаний контент-отдела
- **Owner:** Knowledge Agent
- **Trigger:** новый материал, кейс, шаблон
- **Input:** посты, кейсы, аналитика
- **Steps:** сохранить → тегировать → связать с проектами/клиентами → обновить FAQ
- **Tools:** Obsidian vault
- **Output:** обновлённая база знаний
- **Frequency:** по мере накопления
- **Pain:** контент делается, но не накапливается в базе

## 5. Рекомендованный первый пилот (2-4 недели)

### Цель

Запустить контент-конвейер: **голос → транскрибация → структурирование → черновик поста → адаптация под 1-2 площадки → утверждение человеком → публикация.**

### Что включено (scope)

- 2 площадки для старта: **Telegram** (основной канал) + **X/Twitter** (международный)
- 1 тип сырья: **голосовые заметки Дмитрия** в Telegram
- 2 формата: короткий пост (Telegram) + тред (X)
- Human approval на каждом этапе публикации
- Сохранение всех постов в Obsidian vault
- 1-2 поста в день

### Что НЕ включено на первом этапе

- VK, Dzen, LinkedIn, Instagram (подключаем позже, когда отработан конвейер)
- Визуальный контент (изображения — фаза 2)
- Автоматическая публикация без approval
- Аналитика (ручной сбор первые 2 недели)
- Ответы на комментарии

### Acceptance Criteria (критерии успеха)

- [ ] Голосовое → структурированная заметка в Obsidian за < 5 минут
- [ ] Из одной заметки → минимум 2 формата (Telegram пост + X тред)
- [ ] Дмитрий тратит на контент < 15 минут в день (только наговорить + approve)
- [ ] 10+ постов выпущено за 2 недели
- [ ] Ни одной публикации без approval

## 6. Proposed Agent Contour

### Agent 1: Content Intake Agent

```
Agent name:         Content Intake
Business role:      Приём и структурирование идей
Primary goal:       Превратить голосовую заметку в структурированный брифинг для контента

Inputs:             Голосовые сообщения из Telegram, текстовые заметки
Tools:              Whisper (транскрибация), Telegram API (получение сообщений)
Memory/KB:          Obsidian vault (сохраняет в контент-базу)

Auto actions:       Транскрибировать голос, извлечь тему, ключевые мысли, tone of voice
Approval needed:    Ничего (это внутренний этап)
Forbidden:          Публиковать, отправлять что-либо вовне

Outputs:            Структурированная заметка в Obsidian: тема, мысли, ключевые фразы, tone
Handoff:            → Content Writer Agent
Escalation:         Если голос неразборчив — запросить перезапись
Success metric:     Время от голосового до готовой заметки < 5 мин
Risk:               Ошибка транскрибации → искажение смысла
Human owner:        Дмитрий
```

### Agent 2: Content Writer Agent

```
Agent name:         Content Writer
Business role:      Написание и адаптация контента
Primary goal:       Из структурированной заметки сделать готовые черновики под площадки

Inputs:             Заметка из Obsidian, tone-of-voice, формат площадки
Tools:              LLM (DeepSeek), Obsidian (чтение/запись)
Memory/KB:          Tone-of-voice Дмитрия, прошлые посты, успешные форматы

Auto actions:       Написать черновик, адаптировать под Telegram/X, добавить хэштеги
Approval needed:    Ничего (драфт для утверждения)
Forbidden:          Публиковать без approval, выдумывать факты, обещать от лица компании

Outputs:            Черновики: Telegram пост (до 4096 символов), X тред (до 10 постов)
Handoff:            → Дмитрий (на утверждение), → Content Scheduler (после утверждения)
Escalation:         Если не хватает контекста — запросить у Дмитрия
Success metric:     Дмитрий правит < 20% текста при утверждении
Risk:               Потеря tone-of-voice, галлюцинации фактов
Human owner:        Дмитрий
```

### Agent 3: Content Scheduler Agent

```
Agent name:         Content Scheduler
Business role:      Планирование и публикация контента
Primary goal:       Взять утверждённый пост и опубликовать в нужное время на нужной площадке

Inputs:             Утверждённый пост (текст + площадка + время)
Tools:              Telegram Bot API, X API (xurl)
Memory/KB:          Контент-план в Obsidian, история публикаций

Auto actions:       Добавить в контент-план, опубликовать по расписанию
Approval needed:    Каждая публикация (получает уже утверждённый пост)
Forbidden:          Публиковать без статуса «approved», менять текст после утверждения

Outputs:            Опубликованный пост, запись в контент-плане
Handoff:            → Analytics Agent (после публикации)
Escalation:         Ошибка API → уведомить Дмитрия
Success metric:     100% approved-постов опубликованы вовремя, 0 самовольных публикаций
Risk:               Сбой API, публикация не в то время
Human owner:        Дмитрий
```

### Agent 4: Analytics Agent (фаза 2)

```
Agent name:         Content Analytics
Business role:      Сбор и анализ метрик контента
Primary goal:       Еженедельный отчёт: что залетело, что нет, что улучшить

Inputs:             Метрики из Telegram, X, VK
Tools:              API платформ, Obsidian
Memory/KB:          История метрик

Auto actions:       Собрать цифры, сравнить, дать рекомендации
Approval needed:    Ничего (внутренний отчёт)
Forbidden:          Публиковать аналитику вовне

Outputs:            Еженедельный отчёт в Obsidian: топ-посты, охваты, тренды
Handoff:            → Дмитрий (ознакомление), → Content Writer (улучшение контента)
Success metric:     Отчёт готов каждое воскресенье, содержит actionable рекомендации
Risk:               Неточные данные API
Human owner:        Дмитрий
```

## 9. Risks and Controls

| Риск | Контроль |
|------|----------|
| Агент публикует без approval | Human-in-the-loop: ни один пост не уходит без статуса «approved» |
| Потеря tone-of-voice | Content Writer использует историю постов как референс |
| Галлюцинации фактов | Запрещено выдумывать; если нет данных — запросить у Дмитрия |
| Сбой API при публикации | Scheduler эскалирует ошибку → Дмитрий решает вручную |
| Репутационный ущерб | Никаких обещаний, гарантий, политических высказываний без явного согласования |
