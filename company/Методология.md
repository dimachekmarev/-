## Инфраструктурные принципы

1. Своё железо (Linux/Mac Mini) > облако
2. Одна топовая подписка > десять дешёвых
3. Один качественный агент сначала → клонировать
4. Узкие агенты с фокусированными скилами
5. Профили Hermes + Kanban для оркестрации
6. Codex/gh CLI для починки сломанных агентов
7. Obsidian MD-файлы как память (векторы — оверкилл)

## Безопасная лексика

- ✓ «готовит черновик», «проверяет», «маршрутизирует», «запрашивает approval»
- ✗ «полностью автономный», «заменяет сотрудников», «гарантирует прибыль»

## Топологии агентов

- Single agent — узкий процесс
- Pipeline — стадийная работа
- Supervisor-worker — ветвление по доменам
- Fan-out/fan-in — исследования, аудиты
- Human-in-the-loop — деньги, юристы, клиенты
