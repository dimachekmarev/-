# Единый дневник операций (агенты + проекты)

> Это главный файл дня: сюда попадает всё важное по проектам и агентам.

## Как пользоваться (просто)

1. Каждый день создаём запись в формате `## YYYY-MM-DD`.
2. Внутри дня — 5 блоков:
   - Что сделали
   - Что сломалось
   - Что улучшить завтра
   - Какие решения приняты
   - Оценка качества дня (10-балльная)
3. Ссылки на детальные заметки даём через `[[...]]`.
4. Один день = один короткий отчёт (без воды).
5. Если балл < 7/10 — обязательно пишем причину и план исправления.

- Hermes memory: создан рабочий контур памяти `локальная память -> Obsidian -> GitHub`, поставлены hourly sync (почасовая синхронизация), daily context booster (ежедневное восстановление контекста), Supabase watchdog (контроль Supabase).
- Obsidian memory sync сегодня отработал успешно; локальная память сжата до коротких указателей, полный контекст вынесен в Obsidian.

### Что сломалось / риски
- WordPress-публикация Smart Agent AI не выполнена: отсутствуют `WP_SITE_URL` и `WP_USER`; `WP_APP_PASSWORD` один не даёт отправку через REST API (программный интерфейс сайта).
- Supabase-зеркало памяти подготовлено, но не работает: DNS (адрес проекта) `hagmwnadcvdravmhedcd.supabase.co` недоступен, management token даёт `403`.
- `smart-agentai-autopost-pro` в 14:00 упал с ошибкой Telegram: `failed to get HTTP URL content` — нужен preflight (предпроверка) URL картинки/карточки.
- Вечерний watchdog показал, что до контроля в проектных README и дневнике не было явных операционных фактов за день.

### Что улучшить завтра
- Добавить `WP_SITE_URL` и `WP_USER`, затем прогнать WordPress draft (черновик) и получить `post_id + URL`.
- Починить `smart-agentai-autopost-pro`: проверять HTTP URL до отправки в Telegram и ставить fallback (резервный вариант) без картинки.
- Включить автофиксацию фактов в `Единый дневник операций.md` сразу после каждого боевого cron (запуска по расписанию).

### Решения
- Для вечернего контроля считать внедрением только то, что имеет файл/запуск/статус, а не просто идею.
- Основной источник фактов дня: cron outputs (выходы задач), Obsidian queue (очередь публикации), `hermes cron list`.

### Оценка качества дня (10-балльная)
- Итог: **6/10**
- Что снизило балл: есть рабочие автоматизации и новые контуры памяти, но нет прямой публикации в WordPress, нет клиентского внедрения/денег, часть автопостинга упала.
- Что поднимет балл завтра: закрыть WordPress-доступ, починить автопостинг, начать фиксировать KPI (ключевые показатели) сразу после запусков.
