# Яндекс Диск — файловый агент

## Назначение

Это рабочая схема для отдельного файлового контура Яны по Яндекс Диску.

Яндекс Диск используется как **хранилище файлов**: документы, таблицы, архивы, презентации, исходники, вложения, большие папки.

Obsidian используется как **память и карта знаний**: краткие выжимки, ссылки, структура, решения, что важно и куда смотреть.

## Подключение

- Основной remote: `yandex:`
- Технология: rclone + OAuth Yandex provider
- Проверено: `rclone about yandex:` и верхний список файлов работают
- Индекс: [[yandex-disk-inventory]]

## Как работает процесс

1. Сначала Яна смотрит структуру папки или ссылку, не скачивая всё подряд.
2. Делает карту: какие папки, какие типы файлов, где потенциально важное.
3. В Obsidian заносит только полезную навигацию и выжимки.
4. Сами большие файлы остаются на Яндекс Диске.
5. Если нужно внедрить материалы в работу — Яна предлагает план: что читать, что резюмировать, что перенести в базу знаний.

## Безопасные действия без отдельного подтверждения

- посмотреть список файлов/папок;
- проверить объём;
- найти файл;
- скачать один явно названный файл для анализа;
- создать индекс/карту папки в Obsidian.

## Только после подтверждения Дмитрия

- удалять файлы;
- массово переносить папки;
- перезаписывать файлы;
- запускать синхронизацию с удалениями;
- скачивать большие объёмы на сервер;
- публиковать или делиться файлами.

## Команды-ориентиры

- Проверка доступа: `rclone about yandex:`
- Список верхнего уровня: `rclone lsf yandex: --max-depth 1`
- Индексация: `python3 /root/.hermes/scripts/yandex_disk_inventory.py`

## Стиль отчёта Дмитрию

Коротко:

- **Сделано:** что проверено или найдено.
- **Важно:** что реально полезно.
- **Риск:** если есть удаление, большой объём, приватные файлы или дубли.
- **Дальше:** 1 понятный следующий шаг.
