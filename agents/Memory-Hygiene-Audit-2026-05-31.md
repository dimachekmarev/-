# Memory Hygiene Audit — 2026-05-31

Источник: `agent-memory-hygiene-card-2026-05-31.pdf`
Контур: Hermes default profile

## Что сделано в системе
- Извлечён текст PDF в `.md`.
- Выполнена первичная чистка durable memory по методу Keep / Delete / Rewrite / Limit.
- Зафиксирован Memory Contract для дальнейшей работы.

## Keep (оставить)
- Стабильные предпочтения пользователя по языку/формату общения.
- Устойчивые факты по рабочему контексту (компания, роль, базовый стек).
- Указатель на Obsidian vault как source of truth.
- Технический нюанс доставки файлов в Telegram (ASCII-имена для надёжной отправки).

## Delete (удалить)
Удалены записи, которые нарушают правила гигиены:
1. Запись с хостинг-доступами (содержала логин/пароль).
2. Временный статус «System enhancement: completed/pending».
3. Детальный операционный статус gateway/количества skills (быстро устаревает).

## Rewrite (переписать)
- Переписана процедурная запись в declarative-форму:
  - Было: императивная инструкция как редактировать credentials.
  - Стало: устойчивый факт об ограничении file-tools + необходимость restart gateway для применения конфигов.

## Limit (ограничить область применения)
- Проектное правило «Дикая Клешня: гео Бараново, 22Б; не упоминать Краснодар»
  использовать только в рамках соответствующего проекта, не глобально.
